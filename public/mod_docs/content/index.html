Representing Game Objects

Wherever a game object (Faction, Card, Dongle, Text List, ect.) appears it can be represented in two ways:

  1. Object Query

    An object with some or all of the object type's members. Every object of that type that matches all given members will have the operation applied to them. If no such object exists, a new one will be created, with reasonable defaults given to all missing members.

  2. ID

    An id of a known object. If no object with the given id is found, an ObjectNotFound error will be thrown. An ID with a [previously loaded mod name]: suffix will look for the id as if the offered mod had just been loaded. Be sure to include that mod in your dependencies.

Whenever an array of game objects appear they can be represented in one of two ways:

  1. Array

    List the objects one after the other.

  2. Map

    If an object is provided in place of an array, the object's keys will be used as the value's ids. If the value is a query covering multiple objects, a new object will be created instead. If the key already exists as an id, that id's object will be overwritten with the value's members. If the key already exists but the value is undefined, that id will be removed.

Mass Content Additions

There are two kinds of parametersmembers of a content adding functionobject:
  1. Context parametersmembers
  2. Content parametersmembers

Context parameters appear at the start of the function, members are described here, are named, and describe where the content parametersmembers appear in game. Content parameters appear at the end of the function, members appear in an array under a key the same name as the content adding object, and have no limit to the number added. in one call.

In order to accommodate large content modifications, context parametersmembers can also be specified in between content parametersmembers. To do so, do not set the context parametersmembers to begin with. Instead, wrap the object in another object, with the parameter names as keys, and the object under a non-pluralized content name (eg. card for addCards). wrap the content adding object in an array, and de-pluralize the content key. Then add content parametersmembers as normal.to the encompassing array. Every subsequent content parametermember will use those context parametersmembers until they are overwritten in the same manner.

Something Went Wrong

If your mod throws an unhandled exception it will be treated as if it was never loaded in the first place. Even calls mademembers defined before the error will not apply.

addCards(faction : Faction, additional_faction? : Faction, ...cards : Card[])cards: {faction : Faction, additional_faction : Faction, cards : []}

Cards added by addCardscards will appear as rewards before encounters start, and in shops if they have shop_loc or shop_location set.

Cards with the flagship effect will remove any other previous cards with the flagship effect from the faction. These will not appear as rewards or in shops, but as starting cards.

Cards with the commander effect will not appear in shops.
